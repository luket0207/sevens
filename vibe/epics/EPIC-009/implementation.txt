FEAT-901 to FEAT-911 - Manager Generation and Team Assignment
Status: done

What was implemented:
- Added central manager generation constants:
  - manager tactic pools (reusing existing tactic options)
  - preference count rules.
- Added manager generation utility:
  - procedural preference generation for defensive/attacking tactic sets
  - no preferred/unpreferred overlap per category
  - explicit manager model creator
  - reusable manager preference validator
  - guarded generation loop to prevent invalid outputs.
- Integrated manager creation into AI team generation:
  - each generated AI team now gets one manager at generation time.
- Extended career world totals/debug:
  - added `aiManagerCount` to world totals
  - added manager generation debug summary with:
    - assignment coverage
    - validity counts
    - preference count distributions
    - per-team manager preference details.
- Exposed manager data in career debug views:
  - added manager summary line in calendar debug header
  - added AI Managers JSON card with full preference lists.
- Extended generation progress and competition summaries:
  - added `GENERATING_MANAGERS` progress phase
  - progress now increments once per team manager generated
  - per-competition summary now includes manager count.
- Ensured assignment stability by generating managers only during career generation and persisting directly on team data.

Engine components used:
- `rng` for procedural random manager preference generation.
- `gameContext` for persistence of manager data in career world state.

Engine components not used:
- `bars`, `button`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer` were not required directly for manager generation/assignment logic.
