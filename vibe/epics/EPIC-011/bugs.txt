BUG-001 - Calendar event colour should fill the whole day tile, and icons should be larger

Repro steps
1. Open Career Home calendar view.
2. Go to a day tile with one or more calendar events.
3. Observe event styling in the day tile.

Expected behaviour
- The event colour styling is applied to the full day tile background.
- Event icons are larger and easier to read.

Actual behaviour
- Event colour styling was only applied to small event pods.
- Icons were smaller than intended.

Suspected cause
- Colour classes were applied to `.seasonCalendar__eventIndicator` instead of `.seasonCalendar__day`.
- Icon wrapper dimensions/font-size were set too small for the tile scale.

Attempted fixes
- Added event-derived visual class at day-card level in `seasonCalendar.jsx` using the primary event visual key.
- Moved event colour mapping in SCSS from indicator blocks to day-tile modifier classes.
- Increased event icon wrapper size and calendar event grid slot sizing.

Final fix
- Full day tile now receives event background colour class (`seasonCalendar__day--<eventKey>`).
- Event indicators are icon-only (no coloured pod).
- Icon size increased from 20x20 to 26x26 with larger icon glyph.

Files changed
- `src/game/careerCalendar/components/seasonCalendar.jsx`
- `src/game/careerCalendar/components/seasonCalendar.scss`

Status: fixed
