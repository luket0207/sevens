EPIC-011 - Calendar and Career Flow Tidy Up

Overview
The goal of this epic is to tidy up and improve the usability, clarity, and onboarding flow of the career calendar and surrounding pages.

This epic focuses on:
- making the Continue button smarter and more informative
- replacing calendar tile text with icon-and-colour-based event display
- improving event hover/tooltips
- tidying fixture generation page formatting
- improving small visual details
- removing outdated explanatory text
- correcting skill label contrast
- turning the first day of the career into a forced setup/tutorial day

This epic is about polish, clarity, and stronger flow control.
It is not about adding new competitions or new match systems.

================================================================
Core Goals
================================================================

The main goals of this epic are:

1. Make the Continue button more intelligent
- it should tell the user what action clicking it will perform next

2. Replace calendar tile text entries with icon + colour display
- using Font Awesome icons
- with event-specific tile and icon styling

3. Improve calendar hover behaviour
- match events should show opposition in a tooltip where appropriate

4. Tidy fixture generation page formatting
- cleaner fixture rows
- centered text
- not using HTML lists

5. Apply a set of small UI tidy-ups
- thicker current-day border
- extra Continue buttons on long pages
- remove outdated simulation helper text
- fix player skill label contrast

6. Force first-day team setup as a tutorial-style gate
- the first day should still generate initial fixtures and Champions Cup groups
- but the player must also set up their team and tactics before being allowed to advance to day 2
- the team should begin empty/invalid at career start

================================================================

================================================================
Tooltip Rules
================================================================

If the user hovers over an icon that represents a match:
- show the opposition for that match in a tooltip

Cup match exception:
- if it is a cup match and the player's team is not in that cup, show no tooltip

Cup final exception:
- for cup finals, if the teams are known, show them in the tooltip even if the player's team is not involved

The tooltip should be lightweight, readable, and consistent with the rest of the UI.

================================================================
Continue Button Behaviour
================================================================

The Continue button should become more intelligent.

Instead of generic wording, it should tell the user what will happen next.

Examples:
- Continue to League Fixture Generation
- Continue to Champions Cup Draw
- Continue to Match Day Results
- Continue to Match
- Continue to Next Day
- Continue to Team Setup

The exact wording can vary, but the button must clearly reflect the next forced step in the flow.

This should work for:
- normal day progression
- cup draws
- fixture generation screens
- player match routing
- results screens
- first-day team setup gate

================================================================
First Day Tutorial / Forced Setup Rules
================================================================

The first day of the career should become a tutorial-style mandatory setup day.

Current behaviour to preserve:
- initial fixture generation still happens on day 1
- initial Champions Cup group generation still happens on day 1

New required behaviour:
- when the career starts, the player's team should begin empty and invalid
- the player must not be able to reach day 2 until:
  - they go to the Team Management page
  - place their outfield players into the required positions
  - select their tactics
  - make the team valid
  - save those changes

This means:
- the first day becomes a forced setup day
- team management is mandatory before day 2
- Continue should route the player into the required setup flow if team setup is still incomplete

Important rule
- this is not just a warning or recommendation
- progression to the second day must be blocked until team/tactics setup has been completed and saved

FEAT-1113 - Expand and Fully Fictionalise Player, Team, and Place Name Pools
----------------------------------------------------------------

Description
Make a major update to the naming system used across the game.

This feature expands the available name pools substantially and tightens the fiction rules around place naming so the world feels larger, more varied, and fully invented.

The update applies to:
- player names
- team names
- foreign team names
- place names used anywhere in generated naming such as team names, stadium names, or other location-based identity strings

Requirements
- Substantially increase the amount of names available in the game.
- For all relevant name pools, increase the volume to roughly triple the current amount.
- This applies to all major generated name categories, including:
  - player first names
  - player last names
  - domestic team names
  - foreign team names
  - stadium/place-based naming pools
  - any other shared naming pools currently used by procedural generation
- Keep the naming pools structured, reusable, and easy to expand further later.

Fictional place-name rule
- All place names must now be completely fictional.
- Do not base place names on real cities, towns, countries, regions, or recognisable real-world locations.
- Avoid slight edits of real places that still feel obviously derived from real geography.
- The generated world should feel fully fictional rather than alternate-history real-world.
- This rule applies anywhere place names are used in generated content, including:
  - team names
  - stadium names
  - foreign team identities
  - any location-driven naming systems already in use

Naming quality goals
- Names should still feel believable inside the tone of the game world.
- Player names can still keep the light parody flavour already used in the game, but should feel broader and less repetitive due to the expanded pools.
- Team and foreign team names should feel varied, distinctive, and suitable for football clubs.
- Stadium/place names should feel like real places within the fictional world, without referencing real geography.

Implementation notes
- Prefer curated naming pools over overly chaotic random construction if that gives stronger quality and consistency.
- Keep the pools centralised rather than scattering name fragments across multiple files.
- Review any current naming content and replace entries that are too close to real places.
- The expansion should improve variety and reduce visible repetition across generated careers.
- Keep the naming system efficient and compatible with existing procedural generators.

Acceptance Criteria
- All relevant name pools are expanded to roughly three times their previous size.
- Player name variety is substantially improved.
- Team and foreign team naming variety is substantially improved.
- Place-name-based naming is fully fictionalised.
- No place names are directly based on real-world locations.
- Existing generators use the updated expanded name pools without breaking.
- The generated world feels larger, more varied, and more distinctly fictional.
