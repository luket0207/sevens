FEAT-501 to FEAT-517 - Career Generation, Loading Flow, and Debug Views
Status: done

What was implemented:
- Added full career-generation flow trigger from `/career/start`:
  - replaced direct `to="/career/home"` path with real generation start handler.
  - writes generation queue/progress state into `gameContext`.
  - routes to `/career/generating`.
- Added dedicated loading route/view:
  - new `/career/generating` route and screen.
  - animated spinner + progress bar + phase/detail text.
  - retry path on generation error.
- Added reusable competition schema as single source of truth:
  - League 5: 7 teams, `20..25`
  - League 4: 8 teams, `25..30`
  - League 3: 8 teams, `30..35`
  - League 2: 8 teams, `35..40`
  - League 1: 8 teams, `40..45`
  - Foreign Champions Cup: 12 teams, `42..47`
  - total AI teams = 51.
- Added shared AI team generator utility:
  - generates team identity (name + stadium), kit set, keeper kit, team overall, player targets, role plan, and full players.
- Added curated/template identity pools with uniqueness guards:
  - reusable prefix/suffix pools for team and stadium names.
  - collision handling with retries + fallback labels.
- Reused existing kit model from career-start utilities:
  - `randomiseFullKitSet` for home/away shirts and contrast-safe colours.
  - keeper kit picked from existing keeper option model.
- Added exact-average 7-player target generation per team:
  - integer targets always average exactly to team overall.
  - targets constrained to `teamOverall +/- 5` where possible (and clamped to global range).
- Added enforced slot role/influence assignment:
  - `1 GK`, `2 DEF`, `2 MID`, `2 ATT`.
  - defenders always force `Influence Defender`.
  - midfielders force 50/50 between `Influence Defensive Mid` / `Influence Attacking Mid`.
  - attackers force 50/50 between `Influence Attacker` / `Influence Goalscorer`.
- Reused shared player generation system for all generated players:
  - each slot generates via shared `generatePlayer` path.
  - preserves normal generated fields: skills, overall, potential, influence, name, appearance.
- Added full league/team organization output:
  - stable `competitions` array plus separated `domesticCompetitions` and `foreignCompetitions`.
- Added career data persistence in `gameContext`:
  - expanded `career` state with:
    - `career.generation` (status, progress, timestamps, error)
    - `career.world` (player team + competitions + totals + debug summaries)
  - preserves user-selected setup team and stores it as `career.world.playerTeam`.
- Added real progress tracking across generation phases:
  - preparing
  - generating competitions
  - generating teams
  - generating players
  - finalising
  - saving
  - progress values reflect completed work units.
- Replaced `/career/home` placeholder with nested debug accordion view:
  - league -> team -> players nested `details/summary`.
  - includes team kit renders, overalls, targets, and per-player skill/influence metadata.
  - includes debug distribution JSON for fast verification.
- Added route guards to enforce flow order:
  - `/career/home` redirects to `/career/generating` while generation is running.
  - `/career/home` redirects to `/career/start` if generation is not complete.
- Added debug/test-oriented generation summaries:
  - team overall distribution by competition.
  - per-team player target spread diagnostics.
  - outfield influence distribution counts.

Engine components used:
- `gameContext` for storing setup + generated world + progress lifecycle.
- `bars` for loading progress display.
- `button` for flow actions and retry/back interactions.
- `rng` for team overall rolls, identity variation, and role/influence random decisions.

Engine components not used:
- `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer` (not required for this generation/load/debug slice).

BUG-001 - Career Generation Freezes at 0%
Status: fixed

What was implemented:
- Reduced progress update churn by batching player progress updates to team-level increments.
- Added explicit cancellation support in the generator loop to prevent orphaned background generation work from continuing after unmount/cleanup.
- Added cancellation callback wiring from loading route into generation utility.
- Adjusted progress percent normalization so progress shows at least `1%` once generation has started.
- Increased yielding cadence to once per team to keep the loading UI responsive.
