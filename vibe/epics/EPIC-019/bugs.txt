BUG-001 - Add to Academy confirm flow does not add players
Status: fixed

Repro steps:
1. Open a scouting report with at least one player.
2. Click `Add to Academy`.
3. Confirm in modal and then open Academy page.

Expected behavior:
- Confirming should add the player to academy state.

Actual behavior:
- Player was not added.

Suspected cause:
- Modal used `onYes` callback but defaulted to `OK` button mode, so `onYes` never fired.

Attempted fixes:
- Traced modal API usage in `modalContext` and scouting page.

Final fix:
- Switched Add-to-Academy confirm modal to `MODAL_BUTTONS.YES_NO` so `onYes` executes and the academy add state update runs.
