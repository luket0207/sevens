EPIC-012 - Career Start UX Wizard and Shared Team Setup Layout

Overview
The goal of this epic is to go back over the /career/start page and improve the UX significantly.

The main changes are:
- convert the current career start flow into a wizard-style tabbed setup
- split the setup into clearer stages
- move the first required team selection out of day 1 of the career and into the start flow itself
- allow the user to move freely between setup tabs
- allow the user to start the career from any tab once validation passes
- create a new shared team setup layout/component that can be reused both:
  - on /career/start
  - on the Team Management page

This epic is about:
- better setup flow UX
- improved structure for career start
- reducing friction after career creation begins
- unifying the player-list/pitch layout across setup and team management

This epic is not about:
- changing the core validation rules for starting a career
- changing the underlying player generation rules
- changing the tactic system logic itself
- changing the saved meaning of team selection
- adding new match logic

================================================================
Core Wizard Goals
================================================================

The career start page should no longer feel like one long single page.

Instead, it should become a wizard-style setup with tabs.

Required tab order:
1. Name and Stadium Name
2. Kits
3. Players
4. Team Selection

The user should be able to:
- move freely between tabs
- revisit earlier tabs at any time
- complete setup work in any order after entering the page
- start the career from any tab if the overall setup is valid

Validation rules:
- the same current validation rules for starting a career should still apply
- the Start Career button should only be valid when the full required setup is complete

Important change:
- the first required team selection should no longer happen on day 1 of the career
- it should happen in the career start flow as tab 4 instead
- this means the first-day forced team selection/tutorial gate inside the career must be removed because the setup has already been done before the career begins

================================================================
New Shared Team Setup Layout Component
================================================================

There is currently a similar UI pattern in both:
- the Players step on /career/start
- the Team Management page

Both currently have:
- a player list on the left
- a pitch on the right

This epic should replace that duplicated approach with a shared reusable layout/component.

The new shared layout should behave like this:
- the player list is no longer a vertical list on the left
- the player list becomes a horizontally stacked strip at the top of the pitch area
- each player in the top strip shows only:
  - player image
  - player name
  - overall rating
- player skill sets should no longer be shown in that top player list

Pitch layout changes:
- the pitch should now span the full width of the page beneath the player strip
- the pitch orientation should be rotated conceptually 90 degrees to the right compared with the current view
- this means:
  - goalkeeper on the left
  - defenders to the right of that
  - midfielders to the right of that
  - attackers on the far right

Player details on the pitch:
- once players are placed on the pitch, their full skill sets should be displayed underneath them on the pitch
- each player’s ratings should be arranged in two columns of four
- the overall rating should occupy the top-left slot in this 2x4 layout
- each rating display should show:
  - rating title, left aligned
  - rating bar underneath

This shared component/layout should be reusable in both:
- the career start Team Selection tab
- the Team Management page

Start Career Behaviour
================================================================

The Start Career button should remain governed by the same overall validation requirements as now.

Rules:
- it must still only become valid when all required setup is complete
- the user should be able to click Start Career from any tab
- the button should be available in a consistent location or consistently accessible across tabs

The setup should feel flexible:
- users can jump between tabs
- finish a later step before an earlier one if they want
- revisit and edit before starting
- start as soon as everything is valid

================================================================
Career Flow Impact
================================================================

Because the first required team selection now happens during /career/start:
- remove that requirement from the first day of the career
- the career should no longer force that same setup again after creation
- day 1 should still preserve any other intended day-1 generation/setup behaviour that remains relevant
- only the duplicated team-selection requirement should be removed from the in-career flow

FEAT-1215 - Rebuild Team Management Pitch Component with Formation Layout
----------------------------------------------------------------

Description
Redesign and rebuild the pitch component used for team arrangement so it provides a clearer, more intuitive formation editor.

The current component does not provide the best usability when arranging players, viewing their skill sets, or understanding positional placement.

This feature introduces a new layout and interaction model that improves usability while keeping the existing team logic intact.

The new component must satisfy the following core goals:
- the pitch spans the full width of the page
- all player skill ratings are always visible somewhere in the component
- players can be easily placed into the six outfield positions
- the goalkeeper remains fixed and cannot be moved
- it is always clear which player occupies each position
- rearranging players is quick and intuitive

This component will replace the current pitch layout used in:
- the Team Management page
- the Team Selection tab within the career start wizard

----------------------------------------------------------------
Layout Structure
----------------------------------------------------------------

The new layout should be structured into three main sections:

1. Player Selector Strip
2. Full Width Pitch Formation Area
3. Player Skill Inspector Panel

---------------------------------------------------------------
Player Selector Strip
---------------------------------------------------------------

The top of the component should contain a horizontal strip displaying all available players.

Each player card should contain only:
- player image
- player name
- overall rating

This strip is used for selecting or dragging players onto the pitch.

Full skill sets should NOT be displayed in this strip.

The strip should support:
- dragging players onto the pitch
- clicking players to inspect them

---------------------------------------------------------------
Pitch Formation Area
---------------------------------------------------------------

The pitch should span the full width of the page.

The formation should be displayed from a side-on perspective rather than from behind.

Position order from left to right:

GK → DF → DF → MD → MD → AT → AT

The goalkeeper slot should:
- always contain the goalkeeper
- be locked and not draggable
- clearly display the goalkeeper's identity

Each outfield slot should:
- clearly display the player occupying the slot
- allow players to be dragged into it
- allow swapping if another player is dropped onto an occupied slot

Each slot should be clearly labelled with the position type:
- DF
- MD
- AT

Empty slots should display a placeholder label such as:
- "Empty Defender"
- "Empty Midfielder"
- "Empty Attacker"

---------------------------------------------------------------
Player Skill Inspector Panel
---------------------------------------------------------------

A player skill inspector panel should display the full skill set for the currently selected or hovered player.

This ensures that all player skill ratings are always visible somewhere in the component.

The panel should display:

- player name
- player image
- overall rating
- full skill set

Skill ratings should include:

- Tackling
- Marking
- Passing
- Movement
- Control
- Dribbling
- Shooting

The rating display should use the existing rating bar system.

Each rating should display:
- rating name
- rating bar underneath

----------------------------------------------------------------
Interaction Behaviour
----------------------------------------------------------------

Drag and Drop
- Players can be dragged from the player strip onto pitch slots.
- Dropping a player into an empty slot assigns them to that position.
- Dropping onto an occupied slot swaps the players.

Rearranging Players
- Players already on the pitch can be dragged to another slot.
- Swapping behaviour should be supported between pitch slots.

Removing Players
- Dragging a player off the pitch removes them from that position.

Player Selection
- Clicking or hovering over a player should update the skill inspector panel.

Visual Feedback
- Pitch slots should highlight when a player is dragged over them.
- Occupied slots should clearly show the player assigned to that position.

----------------------------------------------------------------
Goalkeeper Rules
----------------------------------------------------------------

The goalkeeper slot must follow special rules:

- the goalkeeper is always present
- the goalkeeper cannot be moved
- the goalkeeper cannot be swapped with outfield players

----------------------------------------------------------------
Reusability Requirements
----------------------------------------------------------------

The new pitch component should be reusable in both:

- Team Management page
- Career Start Team Selection tab

Shared logic should be centralised so the layout and interactions remain consistent across both pages.

----------------------------------------------------------------
Acceptance Criteria
----------------------------------------------------------------

- A horizontal player selector strip appears above the pitch.
- Player cards in the strip show only image, name, and overall rating.
- The goalkeeper appears in a locked position on the left side of the pitch.
- Six outfield positions can be filled with players.
- Players can be dragged onto the pitch from the selector strip.
- Players can be rearranged between pitch positions.
- Dropping onto an occupied slot swaps the players.
- The position of each player is clearly visible.
- A skill inspector panel displays the full skill set for the selected player.
- All player skill ratings remain accessible within the component at all times.
- The component works in both Team Management and Career Start team selection.
