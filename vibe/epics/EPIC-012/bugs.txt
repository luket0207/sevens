BUG-001 - Cannot drop players onto the new pitch layout
Status: fixed

Repro steps:
1. Open `/career/start` and move to Team Selection.
2. Drag an outfield player from the top strip.
3. Try dropping onto an outfield pitch slot.

Expected:
- Empty slot should behave as a valid dropzone.
- Player should be assigned to that slot.

Actual:
- Drag target behavior was unclear/inconsistent in the new shared pitch layout and appeared to have no usable dropzones.

Suspected cause:
- The shared layout moved away from explicit absolute slot dropzones and relied on a column/grid card layout.
- In Career Start, drag-over prevention was not passed explicitly to the shared layout.

Attempted fixes:
- Restored explicit slot dropzone elements in the shared pitch layout.
- Added clear empty-slot drop affordance (`Drop Player`) and dashed slot styling.
- Passed an explicit `onSlotDragOver` handler in Career Start Team Selection.

Final fix:
- Reworked `TeamSetupLayout` pitch section to absolute-position slot nodes with `onDragOver` + `onDrop` handlers on each slot.
- Added explicit `allowDrop` wiring in `careerStartTeamSelectionTab.jsx`.
