FEAT-1201 to FEAT-1214 - Career Start Wizard + Shared Team Setup Layout Migration
Status: done

What was implemented:
- Reworked `/career/start` into a 4-tab wizard flow with required tab order:
  1. Name and Stadium Name
  2. Kits
  3. Players
  4. Team Selection
- Added free tab navigation with persistent state across tabs.
- Kept `Start Career` available from any tab via a persistent action area.
- Moved the initial required team arrangement/tactics setup into the new Team Selection tab in `/career/start`.
- Added pre-career Team Selection state in `career.setup.teamManagement`, including:
  - slot assignments
  - tactics
  - DTR/ATR/TC snapshot
  - completion timestamp.
- Updated career setup validation so Start Career remains disabled until:
  - name/stadium valid
  - kits valid
  - player selection valid
  - team selection arrangement complete.
- Added a new shared reusable layout component for player arrangement:
  - top horizontal player strip (image, name, overall only)
  - full-width side-on pitch layout (GK -> DF -> MD -> AT left-to-right)
  - full skill display under placed pitch players in a 2x4 grid with bars.
- Reused that shared component in both:
  - `/career/start` Team Selection tab
  - `/team-management` page
- Updated Team Management page to use the shared layout while preserving:
  - drag/drop placement
  - remove-on-off-slot drag end
  - tactic calculation
  - save/cancel flow.
- Removed duplicated in-career day-1 setup gate behavior by disabling day-1 gate activation.
- Updated career world generation to consume pre-career `setup.teamManagement` when complete, so generated player-team state starts with the setup already done.
- Added wizard debug visibility for:
  - active tab
  - overall setup validity
  - team-selector completion
  - team-selection completion
  - current slot/tactic setup payload.

Engine components considered:
- `gameContext`, `bars`, `button`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `rng`, `saveGame`, `timer`.

Engine components used:
- `gameContext` for persisting wizard/tab setup data and pre-career team-selection state.
- `button` for wizard tab controls and start-career actions.
- `bars` in shared pitch rating display cells for the 2x4 rating grid.
- `modal`/`modalContext` remained in use for Team Management save/cancel flows.

Engine components not used (and why):
- `slidePanel`, `toast`, `tooltip`, `makeDecision`, `rng`, `saveGame`, `timer` were not required for this UX-structure migration; existing feature flow did not need new interactions from these systems.
