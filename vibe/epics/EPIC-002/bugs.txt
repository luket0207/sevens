FEATURE - Balance Player Rating Generation System and Stamp Archetypes

Context:
The player generation system already exists and is working, but it is not yet balanced correctly. This feature is to rebalance the existing generation and overall-calculation logic so it produces much more believable football players by position, while still being reversible from skills back into overall rating.

This is not a net-new player generator feature. This is a balancing and structure-improvement feature applied to the existing system.

Main goals:
- Balance generated skill profiles by position.
- Separate generation logic from overall-impact logic.
- Introduce positional archetypes to create realistic variation between players of the same position and same overall.
- Ensure generated players are stamped with their archetype.
- Ensure the calculated overall can still be derived correctly from the generated skill set.
- Apply a one-off special rule that all outfield players must always have Goalkeeping lower than 15.

================================================================
Purpose
================================================================

The current player generation works mechanically, but the outputs are too flat and not believable enough by position.

We want the balanced system to:
- create players that match their position naturally
- allow meaningful variation within the same position and same overall
- allow rounded players where appropriate
- allow some secondary strengths that are not the most important overall drivers
- keep structurally inappropriate skills low on average
- make overall rating depend on role-relevant skills more than irrelevant ones
- remain reversible, so the generated skill set can be used to recalculate the same overall correctly

Examples of intended behaviour:
- Midfielders can sometimes have high Tackling even though Passing/Control/Dribbling/Off Ball matter more to their overall.
- Defenders can occasionally have decent Shooting, but it should usually remain low and should only have a small effect on overall.
- Attackers should vary meaningfully between dribblers, finishers, creator-forwards, runners, etc.
- Goalkeepers should be extremely weak in most outfield skills.
- All outfield players must always have Goalkeeping lower than 15.

================================================================
High-level design change
================================================================

The important balancing change is to split the system into two separate concepts:

1. Generation profile logic
- Controls how likely each skill is to be high or low for a given position.
- Includes positional generation bias and archetype-based variation.
- This is used when generating the skill values.

2. Overall impact logic
- Controls how much each skill contributes to the final overall rating for that position.
- This is used when calculating overall from a finished skill set.

Do not use one simple banding system based only on weight values.
Do not use the overall impact matrix directly as the generation banding system.
The generation system and overall system should be related, but not identical.

================================================================
Variance rules
================================================================

After position bias and archetype bias, apply controlled random variance.

Rules:
- Do not flatten all players of the same overall into near-identical numbers.
- Players of the same position and same overall should still have distinct identities.
- Variance should create trade-offs, not just random noise.

Guidelines:
- Core strengths for the role: around -4 to +4
- Flexible/secondary skills: around -5 to +5
- Suppressed skills: around -4 to +3
- Rare/structurally weak skills: use a low-biased roll instead of normal variance

Important:
- Maintain believable positional shape.
- Midfielders should have the widest flexibility.
- Defenders should usually remain weak at Shooting.
- Attackers should vary strongly across Control, Dribbling, Shooting, Passing, and Off Ball depending on archetype.
- Goalkeepers should remain extremely weak in most outfield skills.

================================================================
Low-biased generation rules
================================================================

Use low-biased generation for structurally weak role skills.

Outfield low-biased roll:
- 85% chance: 18-38
- 12% chance: 39-55
- 3% chance: 56-75

Goalkeeper low-biased outfield roll:
- 88% chance: 12-32
- 10% chance: 33-45
- 2% chance: 46-65

Special exception:
- Do not use the normal outfield low-biased rule for Goalkeeping on DF, MD, or AT.
- Outfield Goalkeeping must always be 1 to 14 inclusive.

================================================================
Overall calculation method
================================================================

Use the overall impact matrix and calculate overall separately from generation.

Step 1: weighted base average

baseOverall =
sum(skill * impactWeight) / sum(impactWeight)

Only include skills where impactWeight > 0.

Step 2: core weakness penalty

Apply a stronger penalty when a player is weak in an important role skill.

Suggested shape:

penalty =
sum(
  ((baseOverall - skill) ^ 1.15) * (impactWeight ^ 1.35) * 0.055
)
for each skill where impactWeight > 0 and skill < baseOverall

Step 3: small role-strength bonus

Apply a small bonus for standout highly relevant role skills.

Suggested shape:

bonus =
sum(
  min(1.2, (skill - (baseOverall + 6)) * 0.03 * impactWeight)
)
for each skill where impactWeight >= 2 and skill > baseOverall + 6

Step 4: final overall

overall = round(baseOverall - penalty + bonus)

Important:
- The calculated overall must be derived from the generated skills, not copied from the original selected rating.
- The generator should then make small corrections as needed so generated players resolve to the intended target overall.

================================================================
Generation flow
================================================================

Rework the existing generator flow so it behaves like this:

1. Choose position
2. Choose target overall
3. Choose an archetype appropriate to the position
4. For each skill:
   generatedSkill = targetOverall + positionBias + archetypeBias + randomVariance
5. For weak-role skills, use low-biased generation instead of normal variance
6. For DF, MD, and AT, always force Goalkeeping to be below 15
7. Clamp all skills to the legal range 1 to 100
8. Calculate overall from the final skill set
9. Apply small corrections to role-relevant skills until calculated overall matches the target overall
10. Preserve the player's archetype identity during correction
11. Save/stamp the selected archetype onto the generated player object

This feature should improve the balance of the current existing generator, not replace it with something disconnected from the current codebase.

- Keep constants/config separated cleanly where sensible.
