BUG-001 - Balance player rating generation and stamp archetypes
Status: done
What was implemented:
- Reworked player generator constants into separate balancing config sets:
  - `POSITION_IMPACT_WEIGHTS` for overall impact only.
  - `POSITION_GENERATION_BIAS` for generation profile only.
  - `POSITION_ARCHETYPES` and `POSITION_ARCHETYPE_BIAS` for positional variation.
  - `LOW_BIASED_SKILL_RULES` and positional low-biased skill mapping.
- Replaced flat weighted generation with a multi-stage pipeline:
  - select archetype for position.
  - generate skills using position bias + archetype bias + controlled variance.
  - apply low-biased roll logic for structurally weak skills.
  - hard-force outfield Goalkeeping in range `1..14`.
  - calculate overall from skills with weighted base + weakness penalty + role-strength bonus.
  - run bounded correction loops to converge generated skills to target overall while preserving archetype identity.
- Updated generated player model to persist `archetype`.
- Updated debug UI to display archetype in generated output.
Engine components used:
- `rng` (`randomInt`) for archetype selection, variance rolls, and low-biased roll bands.
Engine components not used:
- `gameContext`, `bars`, `button`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer` (not required for generator math/config changes; existing integration paths remain unchanged).
Files changed:
- `src/game/careerStart/playerGenerator/playerGeneratorConstants.js`
- `src/game/careerStart/playerGenerator/playerGeneratorUtils.js`
- `src/game/careerStart/components/playerGeneratorDebug.jsx`
Tradeoffs / follow-up tasks:
- Full statistical balancing checks (large sample distribution snapshots by position/archetype) are still manual and should be automated in a future generator test suite.

BUG-002 - Reduce unrealistic high skill spikes at lower target ratings
Status: done
What was implemented:
- Added target-relative skill ceilings in generator/correction bounds so lower-rated players cannot spike into unrealistic high 80s/90s for role-critical skills.
- Added target-aware bias compression so generation bias/archetype bias is scaled down more at lower ratings and scales up toward high target ratings.
- Reworked correction candidate ordering so positive corrections prioritize weaker role-relevant skills before pushing already-high core skills.
- Reduced correction upward caps for high-impact skills to avoid extreme inflation during convergence.
- Softened the core weakness penalty curve constants to reduce over-punishment of weak secondary skills (which previously forced core stats too high to hit target overall).
Engine components used:
- `rng` (`randomInt`) in existing player generation path (unchanged utility dependency).
Engine components not used:
- `gameContext`, `bars`, `button`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer` (BUG-002 is contained to generator math and balancing internals).
Files changed:
- `src/game/careerStart/playerGenerator/playerGeneratorUtils.js`
Tradeoffs / follow-up tasks:
- Balancing values are tuned for realism at the current `70..100` target range and should be revisited when full-squad/statistical automated tests are introduced.
