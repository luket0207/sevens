FEAT-1001 to FEAT-1018 - League/Cup World Simulation and Player Match Debug Flow
Status: done

What was implemented:
- Added a reusable career simulation layer under `src/game/careerSimulation/`:
  - match rating calculation (`(OV * 2) + DTR + ATR + TC`)
  - MRD band resolver and probability matrix outcome selection
  - scoreline generation for draw/normal win/big win
  - league table model, sorting, and result application rules
  - domestic double round-robin fixture generation for League 5 to League 1
  - League Cup state and progression (32-team knockout across 5 rounds)
  - Champions Cup state, seeded group allocation, group simulation, and knockout progression
  - season-end promotion/relegation/playoff/qualification outcome resolver.
- Integrated simulation into calendar state generation:
  - `buildCareerCalendarState` now seeds league/cup simulation state and attaches it to `career.calendar.simulation`.
  - player league fixtures are inserted into season calendar events for visibility.
- Upgraded Career Home progression flow:
  - Continue Day now simulates non-player fixtures on the current day.
  - on player fixture days, day flow routes to `/match` with pending fixture state.
  - added match-day processing loading feedback.
- Replaced placeholder `/match` page with temporary debug resolution page:
  - shows fixture details
  - Home Win / Draw / Away Win actions
  - applies generated scoreline, updates world state, and returns to career flow.
- Added Cups page at `/career/cups`:
  - shows League Cup and Champions Cup structures
  - displays fixtures/results by stage
  - remains inspectable regardless of player participation.
- Added League Table panel to Career Home:
  - default league is player's league
  - player row marked with icon
  - dropdown switch across all domestic leagues
  - full table stat columns.
- Extended debug support:
  - simulation day/fixture logs stored in career calendar simulation debug
  - exposed in Career Calendar debug panel.
- Extended calendar default/reset state shapes in:
  - `gameContext`
  - `careerStart`
  so simulation state persists and resets safely.

FEAT-1022 - Team Form and Home Advantage in Match Simulation
Status: done

What was implemented:
- Added persistent team form support using rolling 5-match result arrays (`W`, `D`, `L`) in simulation state.
- Added form utility module with:
  - form normalization
  - win counting
  - rolling append logic
  - form win-bonus mapping (`0/1/4/7/10`)
  - scoreline-to-form-result mapping.
- Added home-advantage and form-based probability adjustments in match outcome resolution:
  - home team always gets +2% win chance
  - stronger-form side gets mapped bonus based on win-count difference
  - applied after MRD matrix base probabilities
  - distribution rebalanced/normalized to remain valid.
- Team profiles now include form arrays and simulation debug now includes applied form/home modifiers.
- Added automatic form updates after every completed fixture:
  - league fixtures
  - cup fixtures
  - player-forced fixtures
  - season-end playoffs.
- Added team-form initialization/hydration:
  - foreign teams begin with random 5-result form at career generation
  - domestic teams/player team start empty and build naturally
  - old save compatibility via runtime form-map hydration.
- Added cross-season persistence:
  - team form map carried into new season calendar simulation bootstrap
  - season transition now applies form arrays back onto world team objects when rolling seasons.
- Updated default calendar simulation shapes to include `teamFormByTeamId`.
