EPIC-020 - Academy Card Usage and Academy Card Bar

Overview
The goal of this epic is to implement how Academy cards actually work in the game.

This epic introduces:
- Academy card usage from within the Academy page
- an Academy card bar above the academy players
- Academy card handling rules
- maturity reduction cards
- academy slot expansion cards
- academy-value reveal cards for current value and potential
- single-target academy card selection flow for targeted cards
- persistence of newly revealed current/potential values

This epic is about:
- using Academy cards
- applying Academy card effects
- improving the Academy page flow
- integrating Academy cards with coaches' Judgement ratings

This epic is not about:
- changing how Academy cards are generated
- changing the post-match reward system
- changing player generation
- changing scouting report generation
- adding new Academy cards beyond the listed set

================================================================
Career Home Behaviour
================================================================

From the Career Home page:
- all Academy cards should show a button:
  - Go to Academy

This button should:
- simply navigate to the Academy page
- not trigger the card effect directly
- not consume the card
- act only as a shortcut to where the card is used

================================================================
Academy Page Card Bar
================================================================

The Academy page needs a new bar above the academy players.

Requirements:
- this bar holds all Academy cards currently in the player's library
- it should scroll horizontally
- visually it should feel similar to the card library area on Career Home
- it should sit above the academy player list/content

This is the main place Academy cards are used.

================================================================
Academy Card Usage Model
================================================================

From the Academy page card bar:
- the user should be able to use Academy cards

There are 3 usage patterns:

1. Immediate all-player or global effects
- card can simply be used immediately
- effect applies at once

2. Expand Academy special case
- if academy has fewer than 12 slots:
  - add 1 academy slot
- if academy already has 12 slots:
  - this card should instead behave as:
    - Organise Average Friendly Game
    - meaning reduce all academy players' maturity by 3

3. Single-target cards
- cards like:
  - 1 on 1 session
  - Train with the first team
- these do not affect all players
- when one of these is selected:
  - temporarily remove the current academy-player action buttons
  - replace them with a Choose button
  - the user selects which academy player the card applies to
  - after selection/resolution, UI returns to normal

================================================================
Maturity Reduction Card Rules
================================================================

Cards that reduce maturity should apply their effect immediately.

If the card affects all players:
- reduce maturity for all academy players by the specified amount

If the card affects one player:
- reduce maturity for the selected academy player by the specified amount

Maturity floor:
- maturity cannot go below 0

If maturity reaches 0:
- handle it exactly as normal maturity completion is already handled
- this means:
  - show the icon on the Academy button on Career Home
  - update the academy player's available action button state appropriately

================================================================
Current Value and Potential Reveal Cards
================================================================

Cards that help coaches work out player values apply to all academy players.

These cards reveal either:
- current value
- potential value

They do not reveal both unless separate cards are used.

The reveal process uses:
- the coaches' Judgement rating

Use the coaches Judgement rating exactly as described:
- Judgement is out of 100
- this becomes the chance that an academy player is affected by the card

Example:
- coach Judgement = 35
- each academy player independently has a 35% chance to be affected
- if 10 players are in the academy, roughly 3 or 4 would typically be affected

Then for affected players:
- use the card's success chance to decide whether the card succeeds for that player

Success chances by wording:
- slight chance = 25%
- good chance = 50%
- excellent chance = 75%

This second step is also rolled independently per affected player.

If successful:
- the relevant value becomes permanently displayed for that player

Permanent reveal rule:
- if current value is revealed, it stays revealed forever
- if potential value is revealed, it stays revealed forever

Display persistence:
- once revealed, the value should be displayed:
  - in the Academy
  - and later in the first team if that player is promoted

================================================================
Coach Source for Academy Cards
================================================================

These Academy value-reveal cards use the coaches' Judgement rating.

Use the currently active staff/coaches available to the team.
This epic should use the existing team staff state already introduced.

If there are multiple staff members:
- choose a clear consistent rule for which Judgement rating is used
- preferably the best available relevant Judgement rating unless the existing staff system already implies another single-source rule

Keep the rule centralised and explicit in implementation.

================================================================
Card Consumption Rules
================================================================

Academy cards should only be removed from the library when their effect has actually been used successfully.

Remove the card when:
- its effect is applied
- or the targeted academy-player selection has been completed and the effect resolves successfully

Do not remove the card when:
- the player navigates to the Academy page from Career Home
- the user begins a single-target flow but cancels before choosing a player
- the card has not yet been applied
