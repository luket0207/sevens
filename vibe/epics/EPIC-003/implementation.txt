FEAT-410 - Allow Forced Influence Rule in Player Generation Hook
Status: done
What was implemented:
- Added optional forced influence override path through shared generator utilities:
  - `generateOutfieldPlayer({ targetOverall, forcedInfluenceRule })`
  - `generatePlayer({ ..., forcedInfluenceRule })`
  - `generatePlayerBatch({ ..., forcedInfluenceRule })`
- Added influence override resolution and validation utility:
  - accepts forced values:
    - `None`
    - `Influence Defender`
    - `Influence Defensive Mid`
    - `Influence Attacking Mid`
    - `Influence Attacker`
    - `Influence Goalscorer`
  - supports `No Influence` alias internally for compatibility.
  - throws on invalid forced values instead of silently falling back.
- Updated `usePlayerGeneration` hook API to pass optional forced influence through to shared utilities without duplicating rule logic.
- Preserved existing random behavior when no forced rule is supplied.
- Extended `/career/start` debug controls with a `Forced Influence` selector to validate forced-rule flows in-app.
- Persisted forced-rule debug selection in `gameContext` (`career.setup.playerGenerationDebug.forcedInfluenceRule`).
Engine components used:
- `gameContext` for persisted debug setting.
- `button` via existing debug component actions.
Engine components not used:
- `bars`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `rng` (used in generator internals, not in override API plumbing), `saveGame`, `timer`.
