EPIC-014 Implementation Log
Date: 2026-03-10

Scope implemented:
- Replaced the previous EPIC-014 training-first direction with the new roguelike card foundation from the updated feature file.

Features implemented:
- FEAT-1401 / FEAT-1402
  - Added a unified card model and central card config:
    - 4 top-level types (Training, Scouting, Academy, Staff)
    - staff subtypes (Staff Member / Staff Upgrade)
    - rarity model (Common / Uncommon / Rare)
    - central card colour mapping by type.
- FEAT-1403 / FEAT-1404 / FEAT-1405 / FEAT-1406
  - Added fixed definition tables for:
    - Training cards
    - Scouting cards
    - Academy cards
    - Staff upgrade cards
  - Definitions are stored as reusable card data and consumed by the reward system.
- FEAT-1407 / FEAT-1408
  - Implemented procedural staff member generator using rarity + role preference bands.
  - Generator produces:
    - role preference
    - Youth set (Scouting/Judgement/Youth Training)
    - Training set (GK/DF/MD/AT Training)
    - overall rating computed from generated numeric attributes
  - Implemented 50/50 staff split:
    - Staff Member vs Staff Upgrade
    - keeps the originally rolled staff rarity.
- FEAT-1409 / FEAT-1410
  - Added full reward matrix configuration and lookup by:
    - league tier
    - form wins bucket
    - match result
  - Added reward resolver that rolls one weighted type+rarity bucket per call.
- FEAT-1411 / FEAT-1412
  - Added 3 independent reward-roll generation and post-match mandatory choice flow.
  - Added new reward route/page:
    - `/career/card-reward`
  - Match resolution now generates and stores pending 3-card offers, then routes to mandatory choice screen before returning to calendar flow.
- FEAT-1413 / FEAT-1414 / FEAT-1415
  - Added Card Library under the calendar (same calendar panel width).
  - Library supports:
    - horizontal scrolling
    - sorting (name/type/rarity)
    - filtering (type + rarity)
    - discard/removal from library.
- FEAT-1416
  - Added reroll flow in reward-choice page:
    - discard exactly 1 existing library card
    - generate fresh 3-card offer with same reward context
    - still choose exactly 1 resulting reward card.
- FEAT-1417
  - Added debug trigger in calendar debug panel:
    - "Debug Card Reward" button
    - opens normal 3-card reward flow
    - chosen card still follows normal add-to-library path.
- FEAT-1418
  - Added persistent `career.cards` state in game context and career reset flow.
- FEAT-1419
  - Card gameplay effects are intentionally not executed; cards are generation/storage/display/discard/reward-only.
- FEAT-1420
  - Added lightweight debug visibility:
    - reward context
    - matrix row
    - roll outputs
    - staff subtype rolls
    - procedural staff debug
    - reroll count
    - library JSON in debug panel.
