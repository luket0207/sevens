EPIC-022 - Animation Testing Area and PenaltyGoal Prototype

Overview
The goal of this epic is to create an experimental area for building and testing future animations.

This epic introduces:
- a new Animations entry on the home page
- a dedicated Animations page
- a reusable animationContainer component
- a shared football pitch animation base
- a mini kit/player visual for animations
- a simple ball visual for animations
- the first prototype animation:
  - PenaltyGoal
- a reusable goalAnimation end-state component

This epic is about:
- animation testing infrastructure
- reusable animation presentation components
- one working proof-of-concept animation flow

This epic is not about:
- match engine integration
- real match events
- dynamic team data
- user control of the animation outcome
- advanced animation authoring tools

================================================================
Core Goals
================================================================

The main goals of this epic are:

1. Add an Animations entry point from the home page
- below the Start Career button

2. Create an Animations page
- containing an array/list of buttons for triggering test animations

3. Create a reusable animationContainer
- a large modal-like container for playing animations
- it stays open for the animation duration
- it cannot be closed early
- it auto closes at the end

4. Build shared animation visuals
- full football pitch
- mini kit/player markers
- simple ball marker

5. Implement the first animation
- PenaltyGoal

6. Add a reusable goalAnimation component
- shown at the end of goal animations
- used after the PenaltyGoal prototype finishes

================================================================
animationContainer Component
================================================================

Create a new component:
- animationContainer

Purpose:
- acts like a large modal for displaying animations
- fills most of the screen
- overlays above the rest of the UI

Rules:
- takes a value representing how long the animation lasts
- cannot be closed before that duration has elapsed
- closes automatically after that duration
- renders the selected animation as a child inside it

This component should be reusable for all future test animations.

================================================================
PenaltyGoal Animation
================================================================

Create the first animation:
- PenaltyGoal

Scenario:
- home team has a penalty

Initial layout requirements:
- one home kit positioned by the penalty spot
- away goalkeeper positioned in goal
- all remaining players are crowded around the area
- all 14 players must still be visible

Animation sequence:
1. initial setup visible on the pitch
2. stall for a random amount of time between:
   - 0.5 seconds
   - 1.5 seconds
3. the home penalty-taking kit moves towards the ball
4. the ball then moves towards the goal
5. the ball should travel either:
   - to the left of the goalkeeper
   - or to the right of the goalkeeper
6. the left/right destination should be random
7. the ball stops in the goal
8. wait for 0.3 seconds
9. show the goalAnimation component
10. then the animation ends
11. then the animationContainer closes automatically

Important notes:
- this is a goal animation, so the penalty always results in a goal
- no save/miss branch is required in this epic
- the movement can be simple but must clearly show the action

================================================================
goalAnimation Component
================================================================

Create a new component:
- goalAnimation

Purpose:
- displayed at the end of goal animations
- indicates that a goal has been scored

For this epic:
- after the PenaltyGoal animation finishes and the ball is in the net
- show goalAnimation
- then end the overall animation flow

This component should be reusable for other future goal animations.
