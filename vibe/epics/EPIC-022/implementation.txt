Feature implementation pass (2026-03-14, update 1):
- Implemented FEAT-2201 through FEAT-2211 from `EPIC-022/features.txt`.
- Key updates:
  - Added an `Animations` entry below `Start Career` on the home page and wired a new `/animations` route.
  - Created a dedicated Animations page with reusable trigger-card layout and lightweight debug output.
  - Built a reusable `animationContainer` overlay component that stays locked open for the supplied playback duration and auto closes when the animation ends.
  - Built reusable animation presentation components for:
    - full football pitch base with goals and pitch markings
    - mini shirt-based player markers using dummy kit data
    - reusable ball marker
    - reusable end-state `goalAnimation`
  - Implemented the `PenaltyGoal` prototype playback with:
    - random stall between `0.5s` and `1.5s`
    - penalty-taker run-up
    - ball travel to a random left/right finish
    - brief in-goal pause
    - goal end-state overlay
    - automatic container close at total playback end
  - Added lightweight debug support for trigger flow, duration, stall time, shot side, phase progression, and goal-animation display timing.

Tradeoffs / follow-up notes:
- The current animation system uses dummy teams and a fixed prototype scene rather than real match data, which is intentional for this epic.
- `animationContainer` is purpose-built instead of reusing the existing modal because the current modal supports early close via escape/backdrop and does not match the locked playback requirement.
- Manual visual verification is still needed to tune player clustering, motion clarity, and timing feel on both desktop and mobile.
