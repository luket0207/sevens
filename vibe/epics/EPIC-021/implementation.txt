Feature implementation pass (2026-03-14, update 1):
- Implemented FEAT-2101 through FEAT-2115 from `EPIC-021/features.txt`.
- Key updates:
  - Added `Start Session` handling for Training cards on Career Home with coach selection, same-day coach `inUse` locking, Training session creation, and correct card consumption timing.
  - Added persistent first-team training state with sub-ratings per skill and hidden-potential state for first-team players, including normalization during career generation and academy promotion.
  - Replaced the placeholder Training page with a real 3-section Training screen:
    - full first-team player strip
    - focused player details with ratings plus sub-ratings
    - training session panel showing participants, coach details, and session controls
  - Implemented session participant selection from Training card position rules and saved Team Management slot assignments.
  - Implemented OR/PR eligibility gating, coach-type success rolls, exact card-matrix sub-rating gains, visible skill upgrades with no carry-over, and overall recalculation from updated skills.
  - Implemented first-team potential reveal during training using `Judgement / 10`, rounded to 1 decimal place, with a `+5%` bonus for rare cards.
  - Added detailed end-of-session modal reporting and lightweight debug visibility for session setup and resolution data.
  - Updated first-team potential display on Team Management/shared inspector surfaces so hidden potential remains hidden until revealed.

- Implemented `BUG-001` from `EPIC-021/bugs.txt`.
- Root cause:
  - Training participant selection was reading allowed positions from the top-level card shape only.

- What changed:
  - Added a shared `resolveTrainingCardPositions` helper in `src/game/training/utils/trainingSession.js`.
  - Updated participant building to use the shared resolved positions helper.
  - Updated training session snapshot creation to use the same helper so snapshot/debug/session data stay consistent.

Bug fix pass (2026-03-14, update 3):
- Implemented `BUG-002` from `EPIC-021/bugs.txt`.
- Root cause:
  - The OR/PR eligibility resolver was treating `Potential - Overall > 5` as a blocked state.
  - That caused the end-of-session modal to report `Potential gap too large for this session` and incorrectly skipped later training success checks for those players.
- What changed:
  - Updated `resolveEligibilityOutcome` in `src/game/training/utils/trainingSession.js` so `potentialGap > 5` now guarantees eligibility.
  - Cleared the blocked reason for that branch so the session report now shows `Eligible` instead of an exclusion message.
