FEAT-601 to FEAT-610 - Reusable Player Image Component and Team Selector Integration
Status: done

What was implemented:
- Added a reusable `playerImage` feature module:
  - `PlayerImage` render component
  - `parsePlayerAppearance` utility
  - debug gallery helper component for visual verification.
- Added centralized appearance parser/mapping utility for the 3-value appearance array:
  - value 1: skin tone + hair colour mapping (1-5)
  - value 2: head shape mapping (round, wide, tall, square, square tall)
  - value 3: hairstyle mapping (bald, short, short spikey, curly, long)
  - normalizes invalid/non-numeric values into safe `1..5` range.
- Implemented stylised head silhouette and hairstyle variants in `playerImage.scss`:
  - simple South-Park-inspired style direction
  - bald suppresses hair rendering.
- Composed player body from existing team kit renderer:
  - reused `ShirtRenderer` for torso/shirt rendering
  - supports explicit shirt override and default team home kit
  - supports goalkeeper colour kit fallback from existing keeper kit options.
- Added scaling/layout variants (`small`, `medium`, `large`) so player image stays readable in cards/lists.
- Integrated `PlayerImage` into career team selector player displays:
  - choice cards
  - chosen-player list cards.
- Added debug/test support section for image combinations in team selector debug panel:
  - skin/hair values 1-5
  - head shape values 1-5
  - hairstyle values 1-5
  - team-kit variants (home, away, goalkeeper).

Engine components used:
- `button` (existing selector card actions remain via engine button).

Engine components not used:
- `gameContext`, `bars`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `rng`, `saveGame`, `timer` (not required directly for the player-image render/mapping implementation).
