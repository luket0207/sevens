FEAT-401 to FEAT-412 - Career Team Selector End-to-End
Status: done

What was implemented:
- Added a real Career Team Selector flow in `/career/start` with 7 sequential choices:
  - Choice 1: 3 goalkeepers
  - Choices 2-7: 6 outfield groups of 3
- Added up-front pool generation for selector logic:
  - 3 goalkeeper pool (separate pool)
  - 18 outfield pool generated before grouping
- Added weighted outfield target-overall selection for `20..25`:
  - `20` rare
  - `21` common
  - `22` slightly more common
  - `23` common
  - `24` uncommon
  - `25` rare
- Added forced influence spread in outfield generation:
  - Exactly 2 each for:
    - `Influence Defender`
    - `Influence Defensive Mid`
    - `Influence Attacking Mid`
    - `Influence Attacker`
    - `Influence Goalscorer`
  - Remaining 8 outfield players use normal random influence selection.
- Added full-pool shuffle + split into exactly 6 outfield groups of 3.
- Added reusable player card UI for both choice options and chosen-player list.
- Added sequential selection interaction:
  - select one from current group
  - auto-advance to next group
  - no skipping and no multi-select per group through flow design.
- Added persistent chosen-player list with full player details/stats.
- Added selector completion/validation logic:
  - valid only when exactly `1 GK + 6 OUTFIELD` selected.
- Connected selector completion back into Career Start requirement by syncing selected team to `career.setup.players`.
- Persisted full selector state in `gameContext` under `career.setup.teamSelector`.
- Replaced old EPIC-003 generator debug section in Career Start with the new EPIC-004 team selector.
- Added lightweight debug utility (`details` panel) for:
  - goalkeeper pool
  - outfield pool
  - outfield influence spread
  - outfield overall spread
  - chosen state.

Engine components used:
- `gameContext` for persistent career setup/team-selector state.
- `button` for selector actions and per-player selection actions.
- `rng` for weighted overall rolls and pool shuffling.

Engine components not used:
- `bars`, `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer` (not needed for this selector flow implementation).

FEAT-414 - Add Name, Appearance, and Potential to Generated Players
Status: done

What was implemented:
- Extended the shared player generator output so every generated player now includes:
  - `name`
  - `firstName`
  - `lastName`
  - `appearance`
  - `potential`
- Added modular generation utilities:
  - `nameGenerationUtils.js`
  - `appearanceGenerationUtils.js`
  - `potentialGenerationUtils.js`
- Name generation now combines one random first name and one random last name from parody-style pools.
- Appearance generation now returns exactly 3 integers, each in range `1..5`.
- Potential generation now uses final generated overall and applies:
  - `Potential Max = 50 - Current Overall`
  - `Potential = random integer between 1 and Potential Max`
  - edge-safe handling when max is `0` (overall 50) returns `0` without invalid RNG ranges.
- Wired all new generation metadata into the core player factory in `playerGenerationUtils`, so the fields are available everywhere the shared generator/hook is used (including team selector).
- Exposed new generator utilities from `src/game/playerGeneration/index.js` for reuse.
- Updated player display UI to surface generated metadata in selection/debug views:
  - team selector cards now show player name and potential
  - generator debug cards now show name, potential, and appearance

Engine components used:
- `rng` for random name/appearance/potential generation.

Engine components not used:
- `gameContext`, `bars`, `button` (already used by selector shell, not required for generator metadata logic), `modal`, `modalContext`, `slidePanel`, `toast`, `tooltip`, `makeDecision`, `saveGame`, `timer`.
