EPIC OVERVIEW
Last updated: 2026-03-08

EPIC-001 - Core Career Shell (Status: done)
This epic established the first playable structure of the game flow by replacing the landing path with a new Home screen and wiring the main Career routes. It also created the Career Start shell with team setup state for name, stadium, kit, and players. Validation was put in place so Start Career stays disabled until all required setup values exist, even when kit/player sections were still placeholders.

EPIC-002 - Team Kit System and Temporary Generator Layer (Status: done)
This epic delivered the full kit setup foundation, including reusable shirt pattern/config logic, home/away kit editing, contrast enforcement, goalkeeper kit selection, and shirt rendering from saved data. It also introduced and iterated on a temporary player-generation debug path to exercise generation mechanics. Bug work in this epic ended with the temporary generator being removed so the player generation system could be rebuilt cleanly in the next epic.

EPIC-003 - Player Generation Rebuild (Status: done)
This epic replaced the prior generator with a new reusable player-generation module centered on correctness and control. It defined clear GK/outfield skill models, exact-average overall behavior, influence rule selection, transfer logic, and deterministic constraints. The result is a generation API intended for reuse by later career/team systems.

EPIC-004 - Career Team Selector (Status: done)
This epic implemented the staged team-selection flow on Career Start, where the user chooses 1 goalkeeper and 6 outfield players from sequential groups of 3. It added pooled generation rules, forced influence distribution, progression/validation logic, and a persistent chosen-player view with player details. Follow-up bug fixes polished card stat layout and interaction behavior.

EPIC-005 - Career Generation Pipeline and World Build (Status: done)
This epic introduced full career world generation after Start Career, including a dedicated loading route, phased progress, and persistence into game context. It generates league structures, AI teams, kits, and players using prior systems, then transitions to Career Home with debug inspection views. Bug fixes addressed generation stalls/progress reliability and route/flow stability.

EPIC-006 - Reusable Player Image Component (Status: done)
This epic added a reusable visual player renderer driven by appearance arrays plus team kit data. It includes appearance parsing/mapping, stylized head and hairstyle variants, and shirt rendering integration for body visuals. The component was integrated into team selection displays and supported with debug gallery coverage.

EPIC-007 - Career Calendar and Main Loop Foundation (Status: done)
This epic transformed Career Home into the main season loop with a month-based calendar, right-side controls, and debug support. It built season scaffolding, fixture scheduling structure, cup event slots, day-by-day progression, and a new flash modal for day transitions. The work focused on calendar/time structure and scheduling, not match simulation outcomes.

EPIC-008 - Team Management Page (Status: done)
This epic replaced the Team Management placeholder with a full arrangement screen featuring outfield player placement, fixed goalkeeper, tactics controls, and save/cancel flows. It implemented arrangement state, team completeness rules, and tactical rating calculations (DTR/ATR) from grouped skill formulas. Saved team-management state is persisted back into career data and integrated into the career navigation flow.

EPIC-009 - Unspecified / Not Started (Status: not started)
No features, implementation notes, tests, or bug entries are currently defined in this epic folder. It is present as a placeholder for future scope.

EPIC-010 - League and Cup World Simulation (Status: done)
This epic moved Career Home from schedule placeholders to active world progression by adding league tables, fixture simulation, cup state progression, and season-end league outcomes. It introduced a temporary player match resolution page, a day-end results screen, day-1 season fixture reveal flow, and a cup draw resolution flow with game-by-game or full reveal options. The simulation pipeline now persists league/cup results, draw pairings, and debug traces directly in career calendar state for continued play and verification.
